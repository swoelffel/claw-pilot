# i18n — Internationalization of claw-pilot

## Overview

claw-pilot uses `@lit/localize` v0.12.2 in **runtime** mode (dynamic translation loading).
6 languages are supported: English (source), French, German, Spanish, Italian, Portuguese.

## Architecture

```
ui/src/
├── localization.ts          # Central config, initLocale(), switchLocale(), detectLocale()
└── locales/
    ├── en.ts                # English source strings — reference
    ├── fr.ts                # French translations
    ├── de.ts                # German translations
    ├── es.ts                # Spanish translations
    ├── it.ts                # Italian translations
    └── pt.ts                # Portuguese translations
```

### `localization.ts` — key functions

| Function | Role |
|----------|------|
| `initLocale()` | Detect and load locale on startup |
| `switchLocale(locale)` | Change locale + persist to localStorage |
| `detectLocale()` | Priority: localStorage → navigator.language → fallback `en` |
| `allLocales[]` | List of available locales with code, label, flag |

### Automatic detection

1. `localStorage` (`cp-locale`) — user's previous choice
2. `navigator.language` — browser language
3. Fallback: `en`

### Persistence

User's choice is saved to `localStorage` under the key `cp-locale`.

### Language selector

Dropdown `🌐 FR ▾` in the footer of `app.ts`. Shows flags + full names.
The displayed locale is synchronized via the `lit-localize-status` event (not synchronous
on startup — bug fixed).

### Build

Vite generates one JS chunk per locale (e.g., `fr-BvHcxyqc.js` ~3.7kB), dynamically loaded
on demand. No impact on initial bundle.

## Implementation pattern in a component

```ts
// 1. Imports
import { localized, msg } from "@lit/localize";

// 2. Decorator on class
@localized()
@customElement("cp-my-component")
export class MyComponent extends LitElement {

  render() {
    return html`
      <h1>${msg("Page title", { id: "my-component-title" })}</h1>
      <p>${msg("Description", { id: "my-component-description" })}</p>
    `;
  }
}
```

## Add a new language

1. Create `ui/src/locales/xx.ts` by copying `en.ts` and translating the values
2. In `localization.ts`:
   - Add `xx` to `targetLocales`
   - Add import and entry to `localizedTemplates`
   - Add `{ code: "xx", label: "Name", flag: "🏳️" }` to `allLocales`
   - Add `lang.startsWith("xx")` to `detectLocale()`

## Localize a new feature

1. Wrap each UI string with `msg("...", { id: "prefix-id" })`
2. Add `@localized()` decorator to class if missing
3. Add `localized, msg` imports from `@lit/localize`
4. Add corresponding strings at the end of 6 locale files (`en`, `fr`, `de`, `es`, `it`, `pt`)
5. Follow the component ID prefix naming convention

## Points of attention

- **`toLocaleTimeString`**: use `navigator.language` (not hardcoded locale like `"fr-FR"`)
- **Initial locale**: don't read `getLocale()` synchronously at startup — listen to
  the `lit-localize-status` event to ensure locale is loaded
- **DB migrations**: unrelated to i18n, but reminder: always additive (ADD COLUMN nullable)
