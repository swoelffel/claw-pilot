# i18n — Internationalisation de claw-pilot

## Vue d'ensemble

claw-pilot utilise `@lit/localize` v0.12.2 en mode **runtime** (chargement dynamique des
traductions). 6 langues sont supportees : anglais (source), francais, allemand, espagnol,
italien, portugais.

## Architecture

```
ui/src/
├── localization.ts          # Config centrale, initLocale(), switchLocale(), detectLocale()
└── locales/
    ├── en.ts                # Strings source anglaises — reference
    ├── fr.ts                # Traductions FR
    ├── de.ts                # Traductions DE
    ├── es.ts                # Traductions ES
    ├── it.ts                # Traductions IT
    └── pt.ts                # Traductions PT
```

### `localization.ts` — fonctions cles

| Fonction | Role |
|----------|------|
| `initLocale()` | Detecte et charge la locale au demarrage |
| `switchLocale(locale)` | Change la locale + persiste en localStorage |
| `detectLocale()` | Priorite : localStorage → navigator.language → fallback `en` |
| `allLocales[]` | Liste des locales disponibles avec code, label, drapeau |

### Detection automatique

1. `localStorage` (`cp-locale`) — choix precedent de l'utilisateur
2. `navigator.language` — langue du navigateur
3. Fallback : `en`

### Persistance

Le choix de l'utilisateur est sauvegarde dans `localStorage` sous la cle `cp-locale`.

### Sélecteur de langue

Dropdown `🌐 FR ▾` dans le footer de `app.ts`. Affiche drapeaux + noms complets.
La locale affichee est synchronisee via l'event `lit-localize-status` (pas synchrone au
startup — bug corrige).

### Build

Vite genere un chunk JS par locale (ex: `fr-BvHcxyqc.js` ~3.7kB), charge dynamiquement
a la demande. Pas d'impact sur le bundle initial.

## Pattern d'implementation dans un composant

```ts
// 1. Imports
import { localized, msg } from "@lit/localize";

// 2. Decorator sur la classe
@localized()
@customElement("cp-mon-composant")
export class MonComposant extends LitElement {

  render() {
    return html`
      <h1>${msg("Titre de la page", { id: "mon-composant-titre" })}</h1>
      <p>${msg("Description", { id: "mon-composant-description" })}</p>
    `;
  }
}
```

## Ajouter une nouvelle langue

1. Creer `ui/src/locales/xx.ts` en copiant `en.ts` et en traduisant les valeurs
2. Dans `localization.ts` :
   - Ajouter `xx` dans `targetLocales`
   - Ajouter l'import et l'entree dans `localizedTemplates`
   - Ajouter `{ code: "xx", label: "Nom", flag: "🏳️" }` dans `allLocales`
   - Ajouter `lang.startsWith("xx")` dans `detectLocale()`

## Localiser une nouvelle feature

1. Wrapper chaque string UI avec `msg("...", { id: "prefixe-id" })`
2. Ajouter le decorator `@localized()` sur la classe si absent
3. Ajouter les imports `localized, msg` depuis `@lit/localize`
4. Ajouter les strings correspondantes a la fin des 6 fichiers de locale (`en`, `fr`, `de`, `es`, `it`, `pt`)
5. Respecter la convention de prefixe d'ID du composant

## Points d'attention

- **`toLocaleTimeString`** : utiliser `navigator.language` (pas de locale hardcodee comme `"fr-FR"`)
- **Locale initiale** : ne pas lire `getLocale()` de facon synchrone au startup — ecouter
  l'event `lit-localize-status` pour etre sur que la locale est chargee
- **Migrations DB** : sans rapport avec i18n, mais rappel : toujours additif (ADD COLUMN nullable)
