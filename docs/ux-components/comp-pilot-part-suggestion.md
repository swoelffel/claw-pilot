# Suggestion Chips (`cp-pilot-part-suggestion`)

> **Source**: `ui/src/components/pilot/parts/part-suggestion.ts`

Renders follow-up suggestion pills below the last assistant message. Clicking a chip sends the suggestion text as a new user message. Suggestions are generated by the `suggestions` post-middleware after each prompt loop run.

## Mockup

```
┌─ assistant message ──────────────────────────────────┐
│  Here's your quicksort implementation...             │
│                                                      │
│  ┌─ suggestion chips ──────────────────────────────┐ │
│  │  [ Add unit tests ]  [ Optimize for large arrays ]│ │
│  │  [ Explain time complexity ]                     │ │
│  └──────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

## Properties

| Property | Type | Description |
|---|---|---|
| `content` | `string` | JSON-encoded array of suggestion strings |

## Content Format

```json
["Add unit tests", "Optimize for large arrays", "Explain time complexity"]
```

Parsed with `JSON.parse()`, filtered to non-empty strings. Falls back to empty array on parse error.

## Interaction

Click dispatches `CustomEvent("suggestion-click", { detail: { text }, bubbles: true, composed: true })`.

The event bubbles up to `cp-runtime-pilot` which calls `_onSendMessage()` with the suggestion text — same as if the user typed and sent the message.

## Generation

Suggestions are generated by `src/runtime/middleware/built-in/suggestions.ts` (post-middleware, order 90):
1. After prompt loop completes, calls `generateText()` with a lightweight model (`suggestionsModel` config, defaults to agent model)
2. Creates a `suggestion` part on the assistant message
3. Emits `SuggestionsGenerated` bus event → SSE → UI refresh

## Telegram delivery

On Telegram, suggestions are sent as **inline keyboard buttons**. Clicking a button sends the suggestion text as a new inbound message.
